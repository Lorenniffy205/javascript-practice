# Day 2 - JavaScript practice

## Goal

Go deeper into functions and array processing so you can write code that works with real test data.

## What you learn today

- function declarations vs arrow functions
- function expressions and callbacks
- `map`, `filter`, `reduce` for transforming, selecting, and summarizing arrays
- `find`, `some`, `every`, `sort`
- building small reusable helpers for task data
- immutable updates instead of mutating arrays

## Tasks for today

1. Run `node src/day2/day2.js`
2. Read the output in the terminal
3. Open `src/day2/day2.js`
4. Add a new task object to `taskList`
5. Add a helper function using `map`, `filter`, or `reduce`
6. Save and run again

## Practical focus

- turn a list of tasks into readable summaries
- compute totals and counts from arrays
- find specific items by id
- update items without changing the original list
- use functions like real SDET utilities

## Why this matters for SDET

Professional automation is not just click scripts. It is data transformation:

- parse test results
- filter failed cases
- compute metrics for coverage
- summarize execution time

This day shows how to process structured arrays like test case data.
